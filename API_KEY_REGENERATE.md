# Firebase API 키 재생성 가이드

## 방법 1: Firebase Console에서 재생성 (권장)

### 단계별 안내

1. **현재 페이지에서:**
   - 상단 오른쪽의 **⚙️ (설정 아이콘)** 클릭
   - 또는 왼쪽 메뉴에서 **"프로젝트 설정"** 클릭

2. **프로젝트 설정 페이지:**
   - 상단 탭에서 **"일반"** 탭 선택 (기본으로 선택되어 있음)

3. **"내 앱" 섹션 찾기:**
   - 페이지를 아래로 스크롤
   - **"내 앱"** 섹션에서 **"웹 앱"** (gadongonline) 선택

4. **API 키 재생성:**
   - **"웹 API 키"** 섹션 찾기
   - API 키 옆에 **"키 다시 만들기"** 또는 **"재생성"** 버튼 클릭
   - 확인 대화상자에서 **"키 다시 만들기"** 확인

## 방법 2: Google Cloud Console에서 재생성

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 프로젝트 선택: `gadongonline-5da5e`
3. 왼쪽 메뉴 > **"API 및 서비스"** > **"사용자 인증 정보"**
4. **"API 키"** 섹션에서 해당 키 찾기
5. 키 옆 **⋮ (점 3개)** 메뉴 > **"키 다시 만들기"**

## 방법 3: Firebase Console 직접 경로

1. Firebase Console 접속: https://console.firebase.google.com/
2. 프로젝트 선택: `gadongonline-5da5e`
3. 왼쪽 메뉴에서 **⚙️ (톱니바퀴 아이콘)** 클릭
4. **"프로젝트 설정"** 클릭
5. **"일반"** 탭에서 **"내 앱"** 섹션으로 스크롤
6. 웹 앱 선택 후 **"웹 API 키"** 섹션에서 **"키 다시 만들기"** 클릭

## ⚠️ 주의사항

- API 키를 재생성하면 **기존 키는 즉시 무효화**됩니다
- 재생성 후 `js/config.js` 파일을 업데이트해야 합니다
- Vercel 환경 변수도 업데이트해야 합니다
- 배포된 앱이 있다면 즉시 업데이트해야 합니다

## 재생성 후 해야 할 일

1. 새 API 키를 `js/config.js`에 업데이트
2. Vercel 환경 변수 업데이트 (사용 중인 경우)
3. 앱 재배포
4. 테스트하여 정상 작동 확인
